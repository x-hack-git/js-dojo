JavaScriptは動的型付け言語です。動的片付け言語には欠点がありまして、**実行時に「型」を調べるので、実行してから初めて気づく予期せぬエラーが発生してしまう**のでした。

注)**「型」を調べるとは、変数のデータ型**をプログラムがチェックする処理のことです。型チェックとも言います。

例えば、ある関数で数字同士を掛け合わせる処理を記述したとしましょう。この関数を作成したエンジニアは数値が返ってくるのを想定してコードを書いていますが、型の指定が無いので引数に「文字列」や「オブジェクト」など、数値以外のデータを渡すことができてしまいます。

## 用語集
- 動的型付け言語
- 静的型付け言語
- トランスパイル/コンパイル

## 変数に入る値を制限する、関数が返す値を制限する

最初から、square という関数が引数に数値しか取らないようにしておけば、引数に文字列を指定したらプログラムの実行前に「不自然な値が入っている！」と警告を出すことで、書いたプログラムが動かないという**致命的なエラーが防げます**。

Java や C# などの静的片付け言語には型をチェックする仕組みがありますが、JavaScriptは動的片付け言語なのでそれはできませんでした。

そこで、JavaScriptでも静的型付け言語のように「型」があるプログラムを書けるようにと開発されたのが**TypeScript**です。

## トランスパイル/コンパイル

TypeScriptで書いたプログラムはそのままでは実行できないので、実行前に一度**JavaScriptへ変換します(この変換作業をトランスパイル/コンパイル)と呼びます**。

規模の大きなプログラムを複数人で書く場合に、ルールが厳格な方が安全に書ける(ヒューマンエラーを減らせる、ミスに気付きやすくなる)ので、TypeScriptが産み出されました。

## TypeScript はMicroSoftによって開発された言語

TypeScriptはMicroSoftによってメンテナンスされているフリーでオープンソースのプログラミング言語です！ちなみにC#というプログラミング言語もMicroSoftが開発しています。
