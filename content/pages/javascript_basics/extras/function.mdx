## 用語集
- 関数
- パラメータ

- ローカル変数
- 戻り値
- プログラミングの実行の順番左から実行されて、( )の中身が先に実行される

## パラメータ

プログラムの起動と同時に指定し**プログラムの動作を決定する数値や文字**。

- 例) 表計算ソフト「エクセル」を起動する時に、**開くファイルの名前**を指定するのもパラメータ
- 例) 電子レンジで**温め時間**を指定するのもパラメータ

## 関数は小さなプログラム

関数は**サブプログラム**です。プログラムから呼び出し可能な**小さなプログラム**です。

## 関数は処理をまとめたものです

関数を使わなくてもプログラムを書けなくはないです。

では何故、関数というものがプログラミングの世界にあるのかというと、
一連の処理をまとめたものに **「名前」** をつけておけば **「便利だから」** です。

## 内部の処理は同じだけど設定がビミョーに違う時

電子レンジには「温める」という機能がありますが、食品によって温める「時間」が違います。
もし**引数**というアイデアがなければ 「30秒だけ温める機能」「300秒だけ温める機能」のように温める**秒数に応じて沢山の機能を用意しなければなりません**が、「温める秒数は状況によって自由に設定できる」ようにしておくと便利です。

この時、温める機能を実行する際に**引数**として「温め時間」を自由に設定できるようにしておけば**便利です**。

## 引数とパラメータの違い

ちなみに、引数は関数側からみるとパラメータと呼ばれる。呼び出し側からすると引数。

## 引数の名前が変わるのは何故？

```js
// 二乗した数を返す関数
function square(num) {
    return num ** 2
}
　
var number = 10
square(number) // square関数 の num変数(パラメータ) に number:10 を渡して実行している
```

square関数の引数として設定されている `num` は、**変数**です！
この `num` という変数は**square関数内部でしか使えないローカル変数の宣言**です。

こんな書き方もできる！！

```js
var a = 0
square(a=99) // aに99を格納し、square関数のnum変数(パラメータ)にa:99を格納する
```

これもいける！なぜなら **() の中身が先に処理される** からだ！

```js
square(10 + 10 * 2) // square(30)と同じ
```

## 実行の順番

下記のコードを元に解説していきます

```js
var res = square(a=99)
```

コードを実行した時に何が起きているか

```js
// consoleで試そう、必ず試してください
a=99 // この式はa(の結果)が返ります
```

実行の軌跡

```js
square(a=99) // 1. a=99 が実行される
square(a) // 2. a が実行される 99 が返る
square(99) // 3. 99 が num に渡される
```
