---
title: セミコロンについて
slug: "/javascript_basics/extras/semicolon"
---

基本的に、JavaScriptのセミコロンはなくても動作します。ですが、**基本的につけて書いた方がいいです**。セミコロンをつけるつけないは好みの部分もあるのですが、つけない場合に注意することがあります。思わぬバグの原因になる可能性があります。

特に改行を差し込む時に問題が起きやすいです。

## 問題のない書き方

```js
// セミコロンなし
var a
a = 1
console.log(1) // 1
　
// セミコロンあり
var b;
b = 1;
console.log(1); // 1
```

こんなトリッキーな書き方も可能です(実務では避けましょう)
**エラーにならない**というだけで、推奨はされません

```js
var
    a
    =
        100
　
, b =
40
　
console.log(a + b) // 140
```

## 問題のある書き方

return の後のコードは実行されません

```js
function hello() {
    return
    "hello"
}
　
hello() // undefined
```

**予期せぬエラーを避けるために**基本的にセミコロンを書いた方がいいです。

エディタの設定で自動的にセミコロンをつけるようにできますので、そういう機能を活用するとよいでしょう。
